\begin{abstract}
We empirically study scaling laws for decoder-only Transformers trained on SVG (Scalable Vector Graphics) source code. We train five model sizes from $\sim$1M to $\sim$88M parameters under both standard parameterization (SP) and Maximal Update Parameterization ($\mu$P), tune the learning rate on the smallest model, fit a power law $L = a\,N^{-\alpha} + c$ to one-epoch validation loss versus parameter count, and extrapolate to a model $10\times$ wider than our largest. We also generate and quantitatively evaluate SVG samples from the best model.
\end{abstract}

\section{Introduction}
SVG is an XML-based, structured visual language. It has strict syntax (well-formed XML, valid attribute values), hierarchical composition (nested groups, transforms), and renders to images that can be compared visually. This makes it an interesting non-linguistic test bed for studying scaling behavior of language models: prediction quality directly translates to whether the output renders, parses, and looks like a coherent icon.

We replicate, on this domain, the scaling-law methodology of Kaplan et al.\ (2020) \cite{kaplan2020}, augmented with the $\mu$P-based learning rate transfer of Yang et al.\ (2022) \cite{yang2022}. Our contributions are: (1) a clean preprocessing and tokenization pipeline for SVG; (2) a scaling-law fit on SVG data with a comparison of SP versus $\mu$P transfer; (3) extrapolation with bootstrap uncertainty; (4) sample generation and a quantitative validity/render-rate evaluation.

\section{Data}
\paragraph{Sources.} Primary: \texttt{starvector/svg-icons-simple} (HuggingFace, $\sim$89k simplified icons). Supplementary: \texttt{starvector/svg-emoji-simple}, used to expand the corpus toward the 100M-token target. Both come from the StarVector project \cite{rodriguez2023}.

\paragraph{Cleaning.} Each SVG is parsed with \texttt{lxml}; failures are dropped. Comments, processing instructions, \texttt{metadata}, \texttt{title}, and \texttt{desc} subtrees are removed. Whitespace is collapsed and numeric attribute values are rounded to one decimal place to shrink the BPE vocabulary tail. SVGs shorter than 50 characters or longer than 8192 are dropped. Splits are 98/1/1 by file before token concatenation, eliminating leakage.

\paragraph{Tokenization.} A byte-level BPE tokenizer (HuggingFace \texttt{tokenizers}) with vocabulary 4096 trained on the cleaned training split, with special tokens \texttt{<pad>}, \texttt{<bos>}, \texttt{<eos>}, \texttt{<unk>}. Vocabulary 4096 was chosen as a compromise between sequence length (smaller vocab = longer sequences) and embedding cost (larger vocab dominates parameter count for the Tiny model). Each cleaned SVG is wrapped \texttt{<bos>}\dots\texttt{<eos>} and packed into a single uint16 stream; training samples a contiguous \texttt{seq\_len=1024} window.

\subsection{Training Setup}
AdamW ($\beta_1=0.9$, $\beta_2=0.95$, weight decay $0.1$ on 2D weights), cosine learning-rate schedule with 200-step linear warmup decaying to $10\%$ of peak, gradient clipping at $1.0$, batch size of $65{,}536$ tokens per optimizer step, sequence length $1024$. All sizes train for exactly one epoch over the same packed training stream.

\subsection{$\mu$P Parameterization}
Following Yang et al.\ (2022), the $\mu$P version applies: (i) attention scaled by $1/d_{\text{head}}$ rather than $1/\sqrt{d_{\text{head}}}$; (ii) hidden-weight initialization $\mathcal{N}(0, 1/d_{\text{model}})$; (iii) readout (lm\_head) initialized $\mathcal{N}(0, 1/d_{\text{model}}^2)$ with output multiplier $1/d_{\text{model}}$; (iv) per-tensor learning-rate multipliers — readout and hidden weights use $\eta / (d_{\text{model}}/d_{\text{base}})$ while embeddings, norms, and biases keep $\eta$. The proxy width is $d_{\text{base}}=128$ (Tiny). We implement these rules directly in \texttt{model.py} via the \texttt{param\_groups()} helper rather than pulling in the \texttt{mup} package's dynamic re-shape; this keeps the integration explicit and auditable.

\subsection{Learning-Rate Sweep}
We sweep peak learning rate over $\{3{\times}10^{-5}, 10^{-4}, 3{\times}10^{-4}, 10^{-3}, 3{\times}10^{-3}, 10^{-2}, 3{\times}10^{-2}\}$ on the Tiny model under each parameterization, picking the LR with lowest one-epoch validation loss. The selected LR is then transferred unchanged to all larger models in the corresponding curve.

\subsection{Scaling Law}
We fit $L(N) = a\,N^{-\alpha} + c$ (SciPy \texttt{curve\_fit}, bounded). Confidence on the extrapolated loss at $N=10\,N_{\text{XL}}$ uses 2000-sample nonparametric bootstrap over the training points.

\subsection{Evaluation Metrics}
Test-set perplexity, XML validity rate (\texttt{lxml.etree} parses the generated string), SVG render rate (\texttt{cairosvg.svg2png} succeeds), and structural validity (root element is \texttt{<svg>} and a \texttt{viewBox} or width/height pair is present).
